# 위키 작성 규칙 (SCHEMA)

이 위키는 여러 분석 에이전트가 공유하는 단일 상태 저장소다.
모든 에이전트는 아래 규칙을 반드시 지켜 markdown 페이지를 작성한다.

## 1. 근거 기반 작성 (Grounding)
- 모든 정량적 수치는 코드(tools)가 계산해 제공한 값만 사용한다.
- 스스로 숫자를 계산하거나 추정하지 않는다. 제공되지 않은 수치는 "데이터 없음(N/A)"으로 명시한다.
- 등급(A~E)과 위험 신호(RED/YELLOW)는 코드가 확정한 값이며, 절대 변경하지 않는다.

## 2. 페이지 구조
- 각 페이지는 제목(`#`)으로 시작한다.
- 핵심 결론을 먼저, 근거를 다음에 적는다.
- 출처가 있는 수치는 `(출처: 재무 에이전트 계산)`처럼 괄호로 표기한다.

## 3. 불확실성 표기
- 추정이나 해석이 들어간 문장은 "~로 보인다", "~할 가능성이 있다"로 표현한다.
- 확정 사실(코드 계산값)과 해석(LLM 의견)을 문장 단위로 구분한다.

## 4. 금지 사항
- 위키에 없는 다른 기업이나 가상의 데이터를 끌어오지 않는다.
- 투자 권유로 읽힐 수 있는 단정적 표현(예: "반드시 매수")을 피한다.
- 본 분석은 데모이며 실제 투자/대출 판단의 근거가 아님을 전제한다.

## 5. 페이지 목록과 책임
- financials.md : 재무 분석 에이전트 (정량 지표 해석)
- macro.md : 매크로/산업 분석 에이전트
- risk.md : 리스크/공시 분석 에이전트
- overview.md : PM 에이전트 (최종 종합)
